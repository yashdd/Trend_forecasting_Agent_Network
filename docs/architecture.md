# Trend Forecasting Agent Network — Architecture

## Overview

The system detects emerging technology trends 3–5 days before mainstream coverage by analyzing Reddit, Hacker News, and arXiv. A multi-agent pipeline ingests, clusters, scores, validates, and synthesizes signals.

## High-Level Architecture

- **Data layer**: Reddit (PRAW), Hacker News (REST), arXiv (API). Data is fetched daily and stored in PostgreSQL.
- **Backend**: FastAPI serves REST APIs; LangGraph runs the agent pipeline; APScheduler triggers daily and weekly jobs.
- **Storage**: PostgreSQL for relational data; pgvector for embeddings and similarity.
- **Frontend**: React SPA (Vite) with TailwindCSS and Recharts; consumes `/api/v1` endpoints.

## Key Design Decisions

- **Single LangGraph DAG**: One sequential graph (ingestion → embedding/clustering → momentum → cross-source → synthesis) keeps state simple and traceable.
- **Sync pipeline, async API**: Pipeline jobs use sync SQLAlchemy and run in a background thread; FastAPI uses async DB for low-latency reads.
- **Topic identity**: Topics are created or reused by label from BERTopic; assignments are updated per run for documents that were (re)clustered.
- **Weekly reports**: Generated by a separate cron (e.g. Sunday) from the last 7 days of trend insights, written as markdown into `weekly_reports`.

## Scaling Notes

- For larger volume: run ingestion more frequently (e.g. every 6h), full pipeline daily; consider Celery + Redis for job queue.
- Embedding and BERTopic can be run in a worker process; persist the BERTopic model to disk and only transform new documents to reduce cost.
- Use read replicas for dashboard and report APIs if needed.
